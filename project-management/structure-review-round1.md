# 1. 書籍の概要

- 想定読者（`docs/index.md` / `docs/introduction/index.md` の記述ベース）
  - 技術史を「人物の物語」から学びたい一般読者〜エンジニア。
  - 各章のコラム（技術解説/ビジネス教訓）を通じて、現代の開発・組織運営に引き寄せたい読者。
- 本書で達成しようとしているゴール（`docs/index.md` の記述ベース）
  - デジタル革命を支えた革新者の人物像と業績を、背景とともに説明できるようにする。
  - 身近な技術が生まれた経緯をストーリーとして語れるようにする。
  - 技術的ブレイクスルーの意思決定・組織・ビジネス上の示唆を言語化する。

## 3. 構成上の気になる点（候補）

- 目次・章ラインナップの不整合
  - `docs/index.md` と `docs/introduction/index.md` は「10人（組）/全12章」前提で記述されているが、実ファイルと `docs/_data/navigation.yml` には第13章/第14章/第16章が含まれ、章タイトルも一部異なる。
  - 例: `docs/introduction/index.md` の人物リストには「ジョブズ/ベゾス」等が含まれるが、実ファイルの第5章は「UNIX（リッチー&トンプソン）」になっている。
- 章番号の欠番・重複テーマ
  - 第15章が存在せず、番号が飛んでいる。
  - 第10章と第16章がいずれも深層学習/ヒントンを主題としており、章の役割が重複している可能性がある。
- ライセンス表記の不整合
  - `docs/index.md` は “All rights reserved.” だが、`LICENSE.md` は CC BY-NC-SA 4.0（シリーズ統一ライセンス準拠）となっている。どれが正かは `LICENSE.md` を基準に揃えるのが妥当。
- 旧構成の残存（運用面）
  - `cs-visionaries-book-orig/` ディレクトリが存在し、正本がどこかを迷いやすい（要確認）。
